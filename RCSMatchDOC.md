## Общее описание

**RCS** (ранее "Neon Match") — мобильная казуальная игра в жанре Match-3 с киберпанк-тематикой. Игра построена на принципах классических игр Candy Crush и Royal Match, с уникальными механиками комбо-бустеров и системой персонажей с легендарными способностями.

### Жанр
- Match-3 (три в ряд)
- Казуальная мобильная игра
- Киберпанк-эстетика

### Целевая платформа
- Мобильные устройства (iOS/Android)
- Web-браузеры
- Progressive Web App (PWA)

## Архитектура проекта

### MVC паттерн (Model-View-Controller)

#### Model (Модель)
- `Tile.ts` — данные о плитке (цвет, бонус, состояние)
- `Board.ts` — логика игрового поля (матчи, свапы, создание бонусов)
- `SaveSystem.ts` — персистентное сохранение данных

#### View (Представление)
- Все сцены Phaser.js отвечают за рендеринг
- `GameScene.ts` — визуализация игрового поля
- Графические примитивы для отображения плиток

#### Controller (Контроллер)
- Обработка пользовательского ввода
- `onTileClick()` — клики по плиткам
- `attemptSwap()` — логика свапа
- Анимации и переходы

### Паттерн State Machine

Игровое поле использует конечные автоматы для состояний плиток:

```typescript
enum TileState {
  NORMAL = 'normal',    // Обычное состояние
  SELECTED = 'selected', // Выбрана для свапа
  MATCHED = 'matched',   // В составе матча
  FALLING = 'falling',  // Падает вниз
  DESTROYED = 'destroyed' // Уничтожена
}
```

## Игровые механики

### 1. Игровое поле

#### Параметры
- Размер поля: 8x8 (конфигурируемо)
- Размер плитки: 52px (TILE_SIZE)
- Отступы: автоматический расчёт от центра экрана

#### Генерация поля
- Поле генерируется случайным образом
- При генерации проверяется отсутствие начальных матчей
- Каждый цвет имеет равную вероятность появления

### 2. Система цветов плиток

### 3. Формы плиток (Визуальная идентификация)

Каждый цвет имеет уникальную форму для accessibility и визуального разнообразия:

| Цвет | Форма | Особенности |
|------|-------|-------------|
| CYAN | Шестиугольник | 6 вершин |
| PINK | Ромб | 4 вершины, вытянут по вертикали |
| BLUE | Восьмиугольник | Чередующиеся радиусы |
| GREEN | Квадрат со скошенными углами | Срезанные углы 20% |
| YELLOW | Параллелограмм | Наклон 25% |
| PURPLE | Квадрат | Стандартная форма |

### 4. Система матчей (Matching)

#### Определение матча
Матч = 3 или более плиток одного цвета в линию (горизонтально или вертикально).

#### Типы матчей
- **horizontal** — горизонтальная линия
- **vertical** — вертикальная линия
- **l_shape** — L-образная форма (4 плитки)
- **t_shape** — T-образная форма (4 плитки)
- **line_5** — линия из 5 плиток
- **square** — квадрат 2x2

#### Алгоритм поиска матчей
1. Сканирование по горизонтали
2. Сканирование по вертикали
3. Определение формы (L, T, квадрат)
4. Приоритет: 5 плиток > L/T > 4 > 3

### 5. Система бустеров (Boosters)

#### Обычные бустеры

| Тип | Способ создания | Иконка | Эффект |
|-----|----------------|--------|--------|
| ROCKET_H | 4 в ряд (горизонтально) | → | Удаляет всю строку |
| ROCKET_V | 4 в ряд (вертикально) | ↓ | Удаляет весь столбец |
| BOMB | L/T форма | 💥 | Удаляет область 3x3 |
| CROSS | Квадрат 2x2 | ✚ | Удаляет строку + столбец + случайную плитку |
| COLOR_BOMB | 5 в ряд | ★ | Удаляет все плитки случайного цвета (берёт цвет у соседней плитки) |

#### Супер-бустеры (Комбо)

Создаются при свапе двух бустеров:

| Комбинация | Результат | Эффект |
|------------|-----------|--------|
| Rocket + Rocket | SUPER_ROCKET | Строка + Столбец (крест) |
| Rocket + Bomb | SUPER_BOMB | 3 строки + 3 столбца |
| Rocket + Cross | SUPER_CROSS | 3x3 область |
| Bomb + Bomb | MEGA_BOMB | 5x5 область |
| Bomb + Cross | HYPER_BOMB | 4x4 область |
| Cross + Cross | ULTRA_CROSS | Все строки + столбцы |

#### Enum BonusType

```typescript
enum BonusType {

  ROCKET_V = 'rocket_v',
  CROSS = 'cross',

  COLOR_BOMB = 'color_bomb',
  // Супер-бустеры
```

### 6. Логика свапа (Swap Logic)

#### Порядок выполнения свапа

```
1. Анимация свапа (визуальное перемещение)
2. swapTiles() — обмен плиток в данных
3. Проверка комбо (2 бустера на новых позициях)
   → Если ДА: создание супер-бустера (тратит 1 ход)
4. Проверка бустера на новой позиции
   → Если ДА: активация бустера
5. Проверка матчей
   → Если НЕТ: обратный свап (swap back)
   → Если ДА: обработка матчей и трата 1 хода
```

#### Методы Board.ts

```typescript
// Проверка комбо после свапа
checkComboAfterSwap(row1, col1, row2, col2): ComboResult | null

// Создание супер-бустера
createSuperBooster(row1, col1, row2, col2): SuperBoosterResult

// Проверка бустера после свапа  
checkBoosterAfterSwap(row1, col1, row2, col2): BoosterResult

// Активация бустера
activateBonus(bonus, row, col): Tile[]

// Поиск матчей
findMatches(): MatchInfo[]

// Свап плиток
swapTiles(row1, col1, row2, col2): void
```

### 7. Цепная реакция (Chain Reaction)

При активации бустера:
1. Определяются затронутые плитки
2. Проверяются затронутые клетки на наличие бустеров
3. Если бустер найден — добавляется в очередь активации
4. Рекурсивно активируются все связанные бустеры

```typescript
activateBonus(bonus, row, col, visited?): Tile[] {
  // visited предотвращает бесконечные циклы
  const visited = visited || new Set();
  // ...логика активации...
  // рекурсивный вызов для связанных бустеров
}
```

### 8. Падение плиток (Cascade)

После уничтожения плиток:
1. Пустые места заполняются плитками сверху
2. Новые плитки генерируются above the board
3. Анимация падения
4. Проверка новых матчей
5. Повтор до стабилизации

### 9. Система очков

| Действие | Очки |
|---------|------|
| Матч 3 | 100 |
| Матч 4 | 150 |
| Матч 5 | 200 |
| Активация бустера | 100 |
| Активация комбо | 150 |
| Создание супер-бустера | 200 |

## Управление

### Система ввода

#### Клик (Click)
1. **Первый клик** на плитку:
   - Плитка подсвечивается (белая рамка 4px)
   - Сохраняется в `selectedTile`

2. **Второй клик**:
   - Если та же плитка → отмена выбора
   - Если соседняя плитка → попытка свапа
   - Если далёкая плитка → смена выбора

#### Свап (Swap)
```typescript
// Условие: плитки рядом (расстояние = 1)
Math.abs(r1 - row) + Math.abs(c1 - col) === 1
```

#### Режим молотка (Hammer Mode)

**Описание**: Молоток позволяет уничтожить любую выбранную плитку на поле.

**Механика:**
1. Клик на кнопку 🔨 внизу экрана
2. `hammerMode = true` (кнопка подсвечивается красным)
3. Клик на любую плитку → плитка уничтожается с эффектом взрыва
4. Количество молотков уменьшается на 1
5. `hammerMode = false` после использования

**Особенности:**
- Для тестирования количество молотков по умолчанию = 100
- Количество сохраняется в SaveSystem
- Работает независимо от количества (для отладки)

### Обработка состояний

```typescript
// isProcessing блокирует ввод во время анимаций
if (this.isProcessing) return;

// Состояния плиток
- NORMAL: готова к взаимодействию
- SELECTED: подсвечена
- MATCHED: в матче (скоро будет уничтожена)
- FALLING: падает
- DESTROYED: уничтожена
```

### Система целей (Level Goals)

Уровни могут иметь разные типы целей:

| Тип | Описание | Индикация |
|-----|----------|-----------|
| SCORE | Набрать определённое количество очков | Текст "SCORE: X/Y" |
| COLLECT_COLOR | Собрать N плиток целевого цвета | Текст "COLLECT: X/Y" + цветовой круг |

**Цветовой индикатор**: Для COLLECT_COLOR уровней отображается круг с целевым цветом слева от текста цели. Это помогает игроку визуально определить какой цвет нужно собирать.

**Подсчёт collect**: Засчитываются плитки целевого цвета уничтоженные любым способом:
- Обычные матчи
- Молот
- Бустеры (при активации)
- Бустеры (при swap)
- Цепные реакции
- Легендарные способности персонажей

## Известные ограничения

1. **Свайп-управление** — клик-ориентированная механика
2. **Тестирование молотка** — безлимитное количество (для отладки)
3. **Восстановление энергии** — требует интеграции с сервером для real-time
4. **Звуковые эффекты (SFX)** — не реализованы (только музыка)
5. **Анимации персонажей** — только статичные портреты
